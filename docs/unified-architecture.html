Team / Swarm System (swarm/)
1
Leader spawns teammate via Task tool with name + team_name
2
swarm::runner::start_teammate() creates TeammateRunnerConfig, spawns tokio task
3
coordinator::run_teammate_loop() runs: stream turn → go idle → poll inbox → repeat
4
Teammate polls its ~/.claude/teams/{name}/inboxes/{agent}.json for messages
5
SendMessage tool writes to target's inbox via mailbox::write_to_mailbox()
6
Leader polls its inbox via runner::poll_leader_inbox() for teammate replies
~/.claude/teams/{team-name}/ ├── config.json # TeamFile: roster, lead_agent_id, members[] ├── inboxes/ │ ├── team-lead.json # Leader's inbox │ ├── researcher.json # Teammate inbox │ └── verifier.json # Another teammate └── permissions/ # Worker permission sync (plan mode)
Bounty / Economy System (jfc-economy)
1
post_bounty tool → MarketOrchestrator::post_bounty() → bounty enters Open state
2
run_bounty tool → run_bounty_cycle() drives the full solve→validate→settle loop
3
Solver phase: N solvers spawned in parallel git worktrees via SwarmProvider::create_worktree()
4
Each solver runs AgentInvoker::invoke_solver() → produces Solution with patch
5
Validation phase: validators challenge each solution, look for flaws
6
Settlement: settle() ranks solutions, pays winner, updates trust scores
MarketOrchestrator ├── ledger: TokenLedger # Budget tracking, agent balances ├── trust: TrustRegistry # Agent reputation scores ├── bounties: BountyManager # State machine: Open→Bidding→Executing→Validating→Settling→Complete ├── solvers: SolverPool # Active solver agents ├── validators: ValidationPool # Adversarial validators └── charter: Charter # Rules: max_solvers, payment_floor, collusion thresholds

Key integration point: dispatch_heavy.rs implements execute_post_bounty() and execute_run_bounty() which hold the global market_orchestrator() mutex and run the full LLM-backed cycle.
