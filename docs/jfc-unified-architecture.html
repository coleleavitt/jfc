Target Architecture

One registry.
Every agent shape.

The current runtime fragments state across BackgroundTask, InProcessTeammateState, and BackgroundAgentInfo — three overlapping lifecycle authorities. The refactor introduces a single AgentRegistry trait backed by pluggable execution backends, giving every agent type — solo tasks, swarm teammates, economy solvers, council members — a shared identity, status, and message bus.

Execution Backends — AgentRole
InProcessBackend Solo / Foreground

Wraps the current tokio::spawn pattern. Replaces background.rs with a registered agent that reports lifecycle events back into the unified AgentRegistry.

role: Solo lifetime: Foreground replaces: background.rs
TeamBackend Teammate / Persistent

Wraps the swarm mailbox and idle loop. Teammate messages still flow through file-based JSON inboxes, but the identity and status now live in AgentState — not TeammateInfo.

role: Teammate lifetime: Persistent mailbox: MessageBus
DaemonBackend Detached / Background

Wraps the existing BackgroundAgentInfo serialization for cross-process workers. State is persisted to disk and restored by the daemon. AgentId is now UUID-based.

lifetime: Detached replaces: daemon/state.rs
CouncilBackend Council / Parallel

Council is now just a parallel spawn + join. Each model seat is an agent with role: Council. The arbiter synthesis happens after join_all on agent IDs — no separate CouncilSession state.

role: Council lifetime: Foreground arbiter: synthesis
BountyCoordinator Solver / Economy

Economy becomes a coordinator, not an owner. Solvers and validators are spawned through AgentRegistry. Trust score is a field on AgentState. jfc-economy slims to types only.

role: Solver trust_score: i8 worktree: PathBuf
MessageBus Unified Messaging

Single abstraction over in-memory channels (in-process agents) and file-backed JSON inboxes (cross-process/daemon). Replaces the four disconnected messaging paths in the current runtime.

in-memory: mpsc file-backed: inbox_dir

/// Replaces: BackgroundTask, BackgroundAgentInfo, InProcessTeammateState, TeammateInfo
pub struct AgentState {
    pub id:           AgentId,          // Uuid — replaces u64 + String fragmentation
    pub status:       AgentStatus,       // Running | Idle | Completed | Failed | Cancelled
    pub role:         AgentRole,         // Solo | Teammate | Solver | Validator | Council

    // Progress — all agents share this
    pub token_count:  u64,
    pub tool_use_count: u32,
    pub last_tool:    Option<String>,

    // Lifecycle
    pub started_at:   SystemTime,
    pub completed_at: Option<SystemTime>,
    pub error:        Option<String>,

    // Backend-specific (Option, only populated for relevant role)
    pub team_name:    Option<String>,    // Teammate
    pub worktree_path: Option<PathBuf>, // Solver
    pub trust_score:  Option<i8>,       // Economy agents
    pub idle_reason:  Option<String>,   // Teammate
}

pub trait AgentRegistry: Send + Sync {
    fn spawn(&self, config: SpawnConfig) -> AgentId;
    fn status(&self, id: &AgentId) -> Option<AgentStatus>;
    fn state(&self, id: &AgentId) -> Option<AgentState>;
    fn abort(&self, id: &AgentId) -> bool;
    fn complete(&self, id: &AgentId, result: AgentResult);
    fn list(&self) -> Vec<AgentState>;
}
