1. User Message → Stream Response

When you type a message, it flows: TUI input → EngineEvent::UserMessage → stream::start_stream() → Provider::stream() → SSE chunks → AgentTurn accumulation → tool dispatch or text rendering.

2. Task Tool → Subagent Spawn

When the model calls the Task tool with run_in_background: true, the dispatcher spawns a background agent: tool_dispatch → background::spawn() → new tokio task with its own stream loop. Results come back via BackgroundTask completion events.

3. Team / Swarm Architecture

Teams are file-based: ~/.claude/teams/{name}/config.json holds the roster, inboxes/ holds per-agent mailbox JSON files. The leader spawns teammates via swarm::runner::start_teammate(). Teammates poll their inbox, process messages, send results via SendMessage tool → mailbox::write_to_mailbox().

4. Council (Multi-Model Deliberation)

The Council tool fans a question to multiple models in parallel, then an arbiter synthesizes their independent answers. Each council member runs as a direct-mode subagent (no tools, just streaming text). The arbiter sees all responses and produces a consolidated reply.

5. Economy / Bounty Market

The bounty system (jfc-economy) runs competitive solver/validator cycles: post_bounty → spawn N solvers in parallel worktrees → each produces a patch → validators adversarially challenge → settlement ranks winners and distributes trust/tokens.
